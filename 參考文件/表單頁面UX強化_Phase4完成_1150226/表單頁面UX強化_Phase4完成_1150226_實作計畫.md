# 表單頁面 UX 強化 — 實作計畫

## 現況分析

專案共 5 種表單（vendor-work, engineering-work, pending-work, work-report, work-file），各有 `new` 和 `edit` 頁面，共 10 個表單頁面。所有表單使用 react-hook-form + zod + framer-motion + shadcn/ui Card 堆疊式佈局。

### 現有問題
- 紅色 `*` 已存在但缺乏 tooltip 說明為何必填
- 錯誤訊息為小紅字，但只在提交後出現，無即時驗證回饋（綠勾/紅叉）
- 無提交前摘要確認 Dialog
- Header 僅顯示標題，缺乏步驟/進度指示
- 返回按鈕為簡單 icon button，缺乏動畫
- 提交按鈕僅有 loading spinner，無成功動畫

---

## 修改策略

建立 5 個共用元件於 `src/components/forms/` 資料夾，再逐一套用至 10 個表單頁面。

---

## Proposed Changes

### 共用表單元件 (`src/components/forms/`)

---

#### [NEW] [FormField.tsx](file:///Users/user/Desktop/電子白板/whiteboard-nextjs/src/components/forms/FormField.tsx)

**包裝 Label + Input + 即時驗證狀態** 的通用欄位元件

- Props: `label`, `required`, `tooltip?`, `error?`, `touched?`, `children`
- 功能：
  - 必填欄位自動加紅色 `*` + hover 顯示 tooltip（「此欄位為必填」）
  - 欄位被 touch（失焦 `onBlur` 觸發 `trigger(fieldName)`）後根據 error 顯示：
    - ✓ 綠勾（驗證通過） 
    - ✗ 紅叉 + 錯誤訊息（驗證失敗）
  - 使用 framer-motion `AnimatePresence` 讓驗證圖示和訊息淡入淡出

---

#### [NEW] [FormHeader.tsx](file:///Users/user/Desktop/電子白板/whiteboard-nextjs/src/components/forms/FormHeader.tsx)

**表單進度指示 Header** 元件，取代現有 `<header>` 區塊

- Props: `title`, `currentStep`, `totalSteps`, `themeColor`
- 功能：
  - 顯示「步驟 1/3」文字 + 進度條動畫
  - 進度條根據 `currentStep / totalSteps` 自動計算百分比
  - 使用 framer-motion `layoutId` 動畫切換步驟
  - 支援各表單不同的主題色（blue, amber, purple, indigo, teal）

---

#### [NEW] [BackButton.tsx](file:///Users/user/Desktop/電子白板/whiteboard-nextjs/src/components/forms/BackButton.tsx)

**帶動畫返回按鈕**

- Props: `label?`（預設「返回」）, `className?`
- 功能：
  - 左箭頭 + 文字
  - hover 時箭頭向左滑動 `translateX(-4px)` 動畫
  - 使用 `group hover` Tailwind class 實現

---

#### [NEW] [SubmitButton.tsx](file:///Users/user/Desktop/電子白板/whiteboard-nextjs/src/components/forms/SubmitButton.tsx)

**帶成功動畫的提交按鈕**

- Props: `isSubmitting`, `isSuccess`, `label?`, `className?`
- 功能：
  - 三態切換：一般 → loading（spinner）→ 成功（✓ 打勾彈跳動畫）
  - 成功狀態持續 1.5 秒後由外部控制跳轉
  - 使用 framer-motion `variants` 實現打勾縮放彈跳效果

---

#### [NEW] [ConfirmDialog.tsx](file:///Users/user/Desktop/電子白板/whiteboard-nextjs/src/components/forms/ConfirmDialog.tsx)

**提交前摘要確認 Dialog**

- Props: `open`, `onConfirm`, `onCancel`, `title`, `data: Record<string, any>`, `fieldLabels: Record<string, string>`
- 功能：
  - 使用 shadcn `Dialog` 元件
  - 自動以 key-value 列表方式顯示所有已填欄位摘要
  - 空值欄位顯示為灰色「未填寫」
  - 確認與取消按鈕
