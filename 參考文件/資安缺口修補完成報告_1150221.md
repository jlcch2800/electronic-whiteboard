## Fallback 遷移策略

```
登入流程：
1. hash(password) → 嘗試登入
2. 若失敗 → 用明文 password 嘗試
3. 若明文成功 → 自動 updateUser(hash) 遷移
4. 若兩者都失敗 → 真正的密碼錯誤
```

## 使用者待辦

> [!IMPORTANT]
> 以下兩項需要手動完成：

1. **設定 reCAPTCHA 環境變數**（`.env.local`）：
   ```
   NEXT_PUBLIC_RECAPTCHA_SITE_KEY=你的_site_key
   RECAPTCHA_SECRET_KEY=你的_secret_key
   ```
2. **執行 RLS SQL**：在 Supabase Dashboard SQL Editor 中執行 `sql/fix_anon_select.sql`
