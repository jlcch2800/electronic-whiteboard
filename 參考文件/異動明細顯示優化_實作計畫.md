# 異動記錄增強計畫 (Audit Log Enhancement Plan)

## 目標描述
升級 `system_change_log` 以記錄詳細的資料變更（新舊值對照），並確保關鍵的使用者操作（如因密碼錯誤導致的帳號鎖定）可被追蹤。

## 建議變更

### 資料庫架構
#### [修改] `system_change_log` 資料表
- 新增 `old_data` (JSONB) 欄位
- 新增 `new_data` (JSONB) 欄位

### SQL 邏輯
#### [新增] `enhanced_audit_log_migration.sql`
- 用於修改資料表的腳本
- 增強版 `process_system_change_log` 觸發函數：
    - 將 `OLD` 和 `NEW` 該列資料擷取為 JSON 格式。
    - 儲存前移除敏感的 `password_hash`。
    - 針對 `users` 資料表的處理邏輯（雖然通用的 Update 邏輯已可涵蓋鎖定紀錄）。

## 驗證計畫
### 手動驗證
1. 在 Supabase SQL Editor 中執行此 SQL 腳本。
2. 對使用者或廠商施工項目進行編輯操作。
3. 檢查 `system_change_log` 是否有新紀錄，並確認 `old_data` / `new_data` 欄位是否有值。
