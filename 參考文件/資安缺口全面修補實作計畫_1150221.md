## Proposed Changes

### 1. SHA-256 前端預處理 + Key-stretching（H1 + H7）

#### [NEW] [crypto.ts](file:///Users/user/Desktop/電子白板/whiteboard-nextjs/src/lib/crypto.ts)

建立密碼 hash 工具函式：
- `hashPassword(password: string): Promise<string>` — 使用 `crypto.subtle.digest('SHA-256', ...)` 對密碼做 hash，再迭代多次（key-stretching，預設 10000 次）
- 回傳 hex string，作為傳給 Supabase Auth 的「密碼」

#### [MODIFY] [LoginClient.tsx](file:///Users/user/Desktop/電子白板/whiteboard-nextjs/src/app/login/LoginClient.tsx)

呼叫 `signInWithPassword` 前先 `hashPassword(data.password)`

#### [MODIFY] [RegisterClient.tsx](file:///Users/user/Desktop/電子白板/whiteboard-nextjs/src/app/register/RegisterClient.tsx)

呼叫 `signUp` 前先 `hashPassword(data.password)`

#### [MODIFY] [ResetPasswordClient.tsx](file:///Users/user/Desktop/電子白板/whiteboard-nextjs/src/app/reset-password/ResetPasswordClient.tsx)

如果有密碼重設邏輯，也要加 hash

#### [MODIFY] [route.ts](file:///Users/user/Desktop/電子白板/whiteboard-nextjs/src/app/api/admin/users/route.ts)

管理員建立/修改帳號時，**後端也需做相同 hash**（因為管理員從 Admin panel 設定密碼走的是 Server-side Admin API）

## Verification Plan

### 瀏覽器手動驗證

1. **SHA-256 + Key-stretching**
   - 開啟 DevTools Network，登入時觀察 Supabase auth request body，確認密碼欄位是 hex hash（64 字元）而非明文
   - 用已知帳號密碼登入，確認可正常登入（hash 後 Supabase 仍能驗證）

2. **密碼強度**
   - 到 `/register` 輸入僅符合 3 種規則的密碼（如 `Abcdef1!` 少一種），確認可通過驗證
   - 輸入僅 2 種規則的密碼，確認被拒

3. **reCAPTCHA**
   - 開啟登入/註冊/忘記密碼頁面，確認右下角出現 reCAPTCHA badge
   - 提交表單，DevTools Console 確認呼叫 `verify-recaptcha` API 且回傳成功

4. **sessionStorage**
   - 登入後，DevTools → Application → Session Storage 確認有 Supabase session token
   - Local Storage 確認**沒有** Supabase token
   - 關閉瀏覽器視窗後重開，確認需重新登入

5. **ScriptErrors**
   - 在 DevTools Console 手動拋出 `throw new Error('test')`，確認 `system_execution_log` 新增一筆 Error 記錄

6. **RLS anon SELECT**
   - 登出狀態下開啟首頁，確認廠商今日施工項目的數字正確顯示

> [!WARNING]
> SHA-256 前端預處理是一個破壞性變更 — **現有帳號的密碼都是用明文註冊的**，改了 hash 流程後，所有現有帳號都要重設密碼才能登入。
>
> **建議方案**：在 `LoginClient` 中做 fallback — 先嘗試 hash 登入，若失敗則用明文重試一次，並在成功後自動將密碼更新為 hash 版本（migration）。或者，只對新註冊帳號啟用 hash，舊帳號維持原樣直到手動重設。
>
> **請告知想用哪種遷移策略。**
