# 帳號管理功能增強實作計畫

## 目標描述
1.  **UI 欄位擴充**: 在「帳號管理」的編輯/新增對話框中，為管理者新增「失敗計次」、「最後失敗時間」與「鎖定解除時間」的欄位。這將允許管理者查看鎖定狀態，並手動解除鎖定 (清除這些值)。
2.  **密碼變更日誌**: 確保修改密碼時，系統異動紀錄能明確記錄此變更。
3.  **規則釐清**: 確認並回覆使用者的密碼規則疑問 (目前實作為「至少包含 3 種」)。

## 需要使用者審查
> [!NOTE]
> 將在「帳號管理」API 中開放更新鎖定相關欄位，以便管理員解除鎖定。

## 預計變更

### 驗證規則 (Schema)

#### [修改] [schemas.ts](file:///Users/user/Desktop/電子白板/whiteboard-nextjs/src/lib/validations/schemas.ts)
- 更新 `userManagementSchema`，新增以下選填欄位以支援表單處理：
    - `failed_attempts` (number, nullable)
    - `last_failed_at` (string, nullable)
    - `locked_until` (string, nullable)

### 前端元件

#### [修改] [UserManagementClient.tsx](file:///Users/user/Desktop/電子白板/whiteboard-nextjs/src/app/admin/users/UserManagementClient.tsx)
- **UI 更新**: 在 Dialog 表單中新增「帳號狀態」區塊。
    - 顯示 `failed_attempts` (失敗計次)。
    - 顯示 `last_failed_at` (最後失敗時間)。
    - 顯示 `locked_until` (鎖定解除時間)。
    - **新增「解除鎖定」按鈕**: 點擊後將上述三個欄位歸零/清空，這將透過 `setValue` 更新表單狀態。

### 後端 API

#### [修改] [route.ts](file:///Users/user/Desktop/電子白板/whiteboard-nextjs/src/app/api/admin/users/route.ts)
- **PUT (更新使用者)**:
    - 接收 `failed_attempts`, `last_failed_at`, `locked_until` 欄位並更新資料庫。
    - **日誌增強**: 確認當 `password` 有值時，`new_data` 明確包含 `{ password: '***CHANGED***' }`。同時，為確保差異比對 (Diff) 能顯示變更，若 `old_data` 中無 `password` 欄位 (通常沒有)，這已足夠觸發顯示。為求更清晰，可考慮在 `old_data` 注入 `{ password: null }` 或類似標記 (視現有 Diff 邏輯而定，目前邏輯 `undefined` vs `value` 會顯示差異，應已足夠)。

## 驗證計畫

### 手動驗證
1.  **密碼規則確認**:
    - 回覆使用者：目前的程式碼驗證規則為 **「至少包含 3 種」** (大寫、小寫、數字、特殊符號)，不需要全部 4 種。
2.  **解鎖功能測試**:
    - 找到一個已被鎖定 (或手動修改 DB 鎖定) 的帳號。
    - 進入編輯模式，確認看到鎖定時間。
    - 執行解除鎖定 (或手動清空欄位)。
    - 儲存。
    - 確認 `failed_attempts` 歸零，`locked_until` 清除。
3.  **密碼日誌測試**:
    - 僅修改某帳號的密碼。
    - 檢查系統異動紀錄。
    - 確認出現一筆 `Update` 紀錄，且明細中顯示 `password` 的變更。
