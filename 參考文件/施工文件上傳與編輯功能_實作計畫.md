# 實作計畫 - 施工文件上傳 (Cloudinary)

## 目標說明
實作「施工文件」的上傳功能。將原本的文字 URL 輸入框替換為檔案選取按鈕，支援「文件」、「照片」與「影片」上傳。檔案將上傳至 **Cloudinary**，並在列表頁顯示縮圖。

## 使用者審查項目 (User Review Required)
> [!IMPORTANT]
> **Cloudinary 配置**: 此計畫需要使用 Cloudinary 作為圖床。
> 您需要提供以下資訊 (寫入 `.env.local`)：
> 1.  `NEXT_PUBLIC_CLOUDINARY_CLOUD_NAME`: Cloud Name
> 2.  `NEXT_PUBLIC_CLOUDINARY_UPLOAD_PRESET`: Upload Preset (需在 Cloudinary 設定中開啟 Unsigned Upload)
>
> 如果尚未設定，請先至 Cloudinary 後台建立一個 **Unsigned Upload Preset**。

## 預計變更 (Proposed Changes)

### 環境設定
- **`.env.local`**: 新增 Cloudinary 相關變數。

### 前端 (`whiteboard-nextjs`)

#### 1. 新增施工文件頁面 (`src/app/work-file/new/page.tsx`)
- **修改項目**:
    - **移除**：原本的 `Input` 文字輸入框 (URL)。
    - **新增**：`input type="file"` 檔案選取元件。
        - 文件：支援 PDF, DOC, XLS 等。
        - 照片：支援 JPG, PNG 等。
        - 影片：支援 MP4, MOV 等。
    - **邏輯 (createUploadHandler)**：
        - 選擇檔案後，呼叫 Cloudinary API (`https://api.cloudinary.com/v1_1/<cloud-name>/<resource_type>/upload`) 進行上傳。
        - 顯示上傳進度 (Progress Bar)。
        - 上傳成功後，取得 `secure_url` 並填入表單資料。

#### 2. 施工文件列表頁面 (`src/app/work-file/WorkFileClient.tsx`)
- **表格顯示優化**:
    - **文件 (File)**: 顯示連結圖示 (維持不變或優化樣式)。
    - **照片 (Photo)**: 
        - 顯示為 **縮圖** (Thumbnail)。
        - 點擊後開啟原圖。
    - **影片 (Video)**:
        - 顯示為 **影片縮圖** (利用 Cloudinary 自動轉縮圖功能: 影片 URL `mp4` 結尾改為 `jpg` 即可取得縮圖)。
        - 點擊後開啟影片連結。

## 驗證計畫 (Verification Plan)

### 手動驗證
1.  **設定環境**: 確認 `.env.local` 已填入 Cloudinary 資訊。
2.  **上傳流程**:
    - 進入 `/work-file/new`。
    - 各欄位分別選擇一個檔案進行上傳。
    - 確認上傳過程中顯示進度 / Loading 狀態。
    - 提交表單。
3.  **列表檢視**:
    - 進入 `/work-file`。
    - 確認該筆資料：
        - 照片欄位顯示圖片縮圖。
        - 影片欄位顯示影片截圖縮圖。
        - 連結皆正常可開啟。
