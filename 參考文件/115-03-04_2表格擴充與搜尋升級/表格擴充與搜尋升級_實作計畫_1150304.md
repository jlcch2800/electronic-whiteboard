# 實作計畫：表格功能統一升級（空狀態、連續編號、欄位排序）

## 修改策略

> [!IMPORTANT]
> **建議分兩階段進行修改**，以降低出錯風險：
> - **第一階段：修改共用組件**（EmptyState、DataTablePagination 等），可一次生效於所有頁面。
> - **第二階段：逐頁修改各表格頁面**，加入排序功能並修正編號，因為每一頁的欄位、搜尋邏輯各有不同，一次全改容易造成遺漏。

## 修改項目一覽

### 1. 空狀態畫面優化

**現狀**：大多數頁面在無資料的時候只顯示「查無資料」或「沒有找到記錄」等灰色文字，沒有圖示或引導說明。首頁 Dashboard 和部分頁面已使用 `EmptyState` 組件。

**方案**：將所有頁面統一使用 `EmptyState` 共用組件來顯示空狀態，包含：
- 該頁面對應的 lucide 圖示（例如：`Users` 用於廠商、`HardHat` 用於工務）
- 標題文字（例如「今日暫無廠商施工」）
- 描述文字引導使用者操作
- 桌面版表格內的 `<TableRow>` 空狀態、手機版卡片區的空狀態，以及首頁 Dashboard 都統一替換

各頁面空狀態文字：

| 頁面 | 圖示 | 標題 | 描述 |
|------|------|------|------|
| 廠商今日施工 | `Users` | 今日暫無廠商施工 | 目前沒有安排任何廠商施工項目，您可以點擊右上方新增。 |
| 工務今日施工 | `HardHat` | 今日暫無工務施工 | 目前沒有安排任何工務施工項目，您可以點擊右上方新增。 |
| 待處理工作 | `FileClock` | 目前無待處理項目 | 沒有需要追蹤的待處理任務，點擊右上方即可新增。 |
| 施工回報 | `FileText` | 尚無施工回報 | 目前沒有施工回報記錄，您可以點擊右上方新增。 |
| 施工檔案 | `FolderOpen` | 尚無施工檔案 | 目前沒有施工檔案記錄，您可以點擊右上方新增。 |
| 歷史頁面 | 各自對應圖示 | 查無歷史紀錄 | 在選定的日期範圍內沒有找到相關歷史紀錄。 |
| 帳號管理 | `UserCog` | 沒有找到使用者 | 目前沒有符合條件的使用者帳號。 |
| 系統異動紀錄 | `History` | 沒有找到記錄 | 目前沒有符合條件的異動紀錄。 |
| 系統執行紀錄 | `FileText` | 沒有找到記錄 | 目前沒有符合條件的執行紀錄。 |

### 3. 欄位排序功能

**現狀**：專案中已有現成的 `SortableTableHead` 共用組件（位於 `components/ui/sortable-table-head.tsx`），且以下 3 個頁面**已經實作了排序功能**：
- ✅ Dashboard 首頁（廠商、工務、待處理三區塊皆有）
- ✅ 工務今日施工 `EngineeringWorkClient.tsx`
- ✅ 待處理工作 `PendingWorkClient.tsx`

**其餘 10 個頁面尚未實作排序，需逐頁加入。**

排序為前端（client-side）執行，點擊表頭依序循環：升序 → 降序 → 取消排序（恢復預設）。

## 驗證計畫

### 自動化驗證
- 每個階段修改完成後執行 `npm run build` 確認無編譯與型別錯誤。

### 手動驗證
1. 逐頁檢查空狀態畫面是否呈現圖示、標題與說明文字。
2. 確認所有頁面的序號為跨頁連續累加且在手機版卡片也一致。
3. 點擊表頭驗證排序功能（升序 → 降序 → 取消）是否正確。
4. 切換至手機版確認排版不會跑版。
