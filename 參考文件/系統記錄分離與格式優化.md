# 系統記錄分離與格式優化

本次更新包含兩個部分：資料庫層面的記錄分離（防止「系統執行記錄」被使用者操作污染）與前端介面的繁體中文格式化。

## 1. 系統異動記錄 (System Change Log) 格式優化

已更新 `src/app/admin/change-log/ChangeLogClient.tsx`，達成以下改善：

-   **排除系統自動記錄**：預設排除 `System` 帳號的自動操作記錄，確保介面只顯示使用者的實際操作。
-   **全站繁體中文**：
    -   **資料表名稱**：如 `pending_work` 顯示為 `待處理工作項目`。
    -   **欄位名稱**：如 `vendor_name` 顯示為 `廠商名稱`。
    -   **動作類型**：如 `Insert` 顯示為 `新增`。
-   **明細顯示優化**：異動明細彈窗現在採用與系統執行記錄相同的排版：
    -   上方顯示基本資訊列表。
    -   下方顯示「異動資料對比」表格，並以綠色/紅色區分變更前後的數值。
-   **匯出優化**：Excel 匯出（包含清單與明細）現在與「系統執行記錄」的格式一致，採用中文欄位並依照指定順序排列。

## 2. 資料庫觸發器調整 (Database Triggers)

為了讓「系統執行記錄」只包含排程 (Cron Job) 的執行結果，而不包含使用者的 CRUD 操作，我們需要移除舊的觸發器。

已在您的專案中建立 SQL 檔案：`sql/drop_execution_triggers.sql`

請至 Supabase Dashboard 的 SQL Editor 執行此檔案內容，或複製以下指令執行：

```sql
-- 移除自動寫入系統執行記錄的觸發器
DROP TRIGGER IF EXISTS log_execution_trigger ON vendor_today_work;
DROP TRIGGER IF EXISTS log_execution_trigger ON vendor_today_work_history;
DROP TRIGGER IF EXISTS log_execution_trigger ON engineering_today_work;
DROP TRIGGER IF EXISTS log_execution_trigger ON engineering_work_history;
DROP TRIGGER IF EXISTS log_execution_trigger ON pending_work;
DROP TRIGGER IF EXISTS log_execution_trigger ON pending_work_history;
DROP TRIGGER IF EXISTS log_execution_trigger ON work_report;
DROP TRIGGER IF EXISTS log_execution_trigger ON work_report_history;
```

執行後，「系統執行記錄」將不再自動記錄一般資料表的增刪改，僅會記錄系統排程的主動回報。

## 驗證步驟

1.  **檢查異動記錄**：前往「系統異動記錄」頁面，確認不再看到此前的 "System" 記錄，且所有欄位皆為中文。
2.  **執行 SQL**：執行上述 SQL 指令。
3.  **測試分離**：
    -   以使用者身份新增一筆資料（例如待處理工作）。
    -   確認該筆操作**出現在**「系統異動記錄」。
    -   確認該筆操作**不出現在**「系統執行記錄」。
