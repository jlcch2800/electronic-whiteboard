# 實作計畫 - 分離與格式化系統記錄

## 目標描述
1.  **分離記錄**：
    -   `system_execution_log` (系統執行記錄)：停止儲存資料表的增刪改操作。僅保留排程 (Cron Job) 的執行結果 (由排程主動呼叫記錄函數)。
    -   `system_change_log` (系統異動記錄)：確保僅顯示使用者的操作 (過濾掉 'System' 的自動更新)。
2.  **統一介面標準**：
    -   將 `ChangeLogClient` (異動記錄介面) 中的所有資料表名稱與欄位翻譯為繁體中文。
    -   讓異動記錄的匯出格式與執行記錄保持一致。

## 變更內容

### 資料庫 (Database)
#### [NEW] [drop_execution_triggers.sql](../sql/drop_execution_triggers.sql)
-   移除 `vendor_today_work`、`engineering_today_work` 等資料表上的觸發器 (Triggers)，這些觸發器原本會呼叫 `log_execution_with_details`。
-   這確保了只有明確呼叫 (如 Cron jobs) 才會寫入執行記錄，達成記錄分離。

### UI 元件 (UI Components)

#### [MODIFY] [ChangeLogClient.tsx](file:///Users/user/Desktop/電子白板/whiteboard-nextjs/src/app/admin/change-log/ChangeLogClient.tsx)
-   **翻譯 (Translations)**：
    -   新增 `TABLE_NAME_MAP`、`FIELD_LABELS`、`FIELD_ORDER` (參照 `ExecutionLogClient` 的定義)。
    -   將翻譯應用於表格列表與明細彈窗。
-   **過濾 (Filtering)**：
    -   更新 `filteredLogs` 邏輯，排除 `user_account` 為 'System' (或 `name` 為 'System') 的記錄。
-   **匯出 (Export)**：
    -   更新 `handleExport` 使用中文標題 (`單位`、`姓名`、`動作類型`、`異動資料表` 等)。
    -   更新 `handleExportDetail` 使用 `FIELD_ORDER` 排序與中文欄位標籤，確保與 `ExecutionLogClient` 的結構一致。

## 驗證計畫
1.  **系統執行記錄 (Execution Log)**：
    -   執行資料表更新 (例如：新增一筆待處理工作)。
    -   驗證該操作 **不會** 出現在「系統執行記錄」中。
    -   (排程執行的結果仍應出現)。
2.  **系統異動記錄 (Change Log)**：
    -   驗證 "System" 的記錄已不再顯示。
    -   驗證資料表名稱顯示為中文 (例如：顯示 `待處理工作項目` 而非 `pending_work`)。
    -   驗證匯出的 Excel 檔案包含中文標題且格式正確。
