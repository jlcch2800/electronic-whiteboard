# 實作計畫：統一修改分頁功能

這份計畫的目標是改善「所有擁有分頁功能」的網頁，讓使用者能更彈性地跳轉至特定頁面，並確實隱藏在手機版介面下的「第一頁/最後一頁」箭頭。

## 回答您的問題：是否要分次修改？
**建議一次修改所有網頁。** 
在我們的專案架構中，所有表格的分頁功能都已經共用同一個核心組件：`DataTablePagination.tsx`。
這意味著，**我們不需要逐一修改每一個網頁**。只要修改這單一個核心組件，所有的網頁（包含派工單、工作紀錄、歷史紀錄、帳號管理等）都會自動且一致地獲得這項新功能。這種做法最安全、最有效率，且最不會出現漏改的錯誤。

## 預計修改項目

### [DataTablePagination.tsx] 修改核心的分頁組件
#### [MODIFY] [DataTablePagination.tsx](file:///Users/user/Desktop/電子白板/whiteboard-nextjs/src/components/DataTablePagination.tsx)

1.  **實作「選擇特定頁面」**：
    將原本單純顯示「第 X 頁，共 Y 頁」的文字，重構為一個下拉選單（使用專案內的 `Select` 組件）。使用者點擊目前的頁數時，會展開所有可選頁數的列表（1, 2, 3... 到整體最大頁數），點擊特定數字即可直接跳轉。若總頁數為 0 則該選單會自動停用。
2.  **手機版確保顯示第一頁/最後一頁圖示**：
    目前的原始碼中使用了 `<Button className="hidden lg:flex ...">` 將第一頁和最後一頁按鈕在小螢幕上隱藏。依據您的新需求，我們將移除這些 `hidden` 和 `lg:flex` 的 CSS 類別，確保不論在桌面或手機版，這四個導航箭頭（第一頁、上一頁、下一頁、最後一頁）都會完整顯示。

### [VendorWorkClient.tsx] 修復廠商今日施工頁面缺失的分頁
#### [MODIFY] [VendorWorkClient.tsx](file:///Users/user/Desktop/電子白板/whiteboard-nextjs/src/app/vendor-work/VendorWorkClient.tsx)
經過檢查 `VendorWorkClient.tsx` 的程式碼，發現這個頁面**完全沒有匯入並使用 `DataTablePagination` 組件**，同時也缺少了控制頁碼 (`currentPage`) 與每頁筆數 (`itemsPerPage`) 的狀態管理機制。
為了解決這個問題，我將會：
1. 為 `VendorWorkClient` 加上 `currentPage`, `itemsPerPage`, `totalPages` 等分頁相關的狀態 (state) 。
2. 匯入共用的 `<DataTablePagination />` 組件並放置於表格/卡片列表的下方。
3. 修改讀取資料 (`refreshData`, `handleSearch`) 和畫面渲染的邏輯，使其能夠根據目前頁數切換顯示。

## 驗證計畫

### 手動驗證測試
實作完成後，請您協助進行以下手動驗證：
1. 本機將已經啟動開發伺服器 (`npm run dev`)。
2. 前往任何一個包含大量資料的頁面（例如：管理員的「帳號管理」或是「工作紀錄」）。
3. **驗證選擇特定頁數**：在網頁底部的分頁控制區，點擊目前的頁碼，嘗試從選單中選擇其他特定頁數（例如第 2 頁），確認表格資料是否有更新以及頁碼標籤是否正確變更。
4. **驗證手機版體驗**：使用瀏覽器的開發者工具切換成手機預覽模式（或是將視窗拉窄），確認分頁區域最左邊與最右邊的標籤（到第一頁和到最後一頁的按鈕）**可以正常顯示且排版不會跑掉**。
5. **驗證廠商今日施工頁面**：前往「廠商今日施工項目」，確認下方已經成功顯示分頁功能，並能正確切換與計算筆數。
