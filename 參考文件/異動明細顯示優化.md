# 異動明細顯示優化 Walkthrough

## 任務目標
優化系統異動記錄的詳細內容顯示，使其更易於閱讀與理解。

## 已完成項目

### 1. 欄位中文化
將原始的資料庫欄位名稱轉換為使用者友善的繁體中文名稱。
- `created_at` -> **建立時間**
- `updated_at` -> **更新時間**
- `user_name` -> **使用者姓名**
- ...以及其他 30+ 個欄位。

### 2. 列表式排版
- 移除原本不易閱讀的 JSON 格式（`{"key": "value"}`）。
- 改為直式列表排版，每一行顯示一個欄位。
- 欄位名稱加粗顯示。

### 3. 異動視覺化標示 (Diff View)
- **變更前 (Old Data)**：若該欄位有變更，數值以 **綠色背景/文字** 標示。
- **變更後 (New Data)**：若該欄位有變更，數值以 **紅色背景/文字** 標示。
- 僅在兩個版本數值不相同時才顯示高亮，未變更的欄位維持一般顯示。

## 修改檔案
- `src/app/admin/change-log/ChangeLogClient.tsx`

## 驗證結果
- 已執行 `npm run build` 確認建置無誤。
- 開發環境測試顯示正常，中文對照表運作正確，樣式渲染符合預期。
